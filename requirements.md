# Требования CRM Gamix

> Черновик для фиксации требований. Заполните фактические данные из обсуждения, если они уже есть.

## 1. Роли и пользователи
- Администратор: управление пользователями, ролями, справочниками.
- Руководитель отдела продаж: контроль воронки, отчёты, SLA.
- Менеджер по продажам: работа с лидами/сделками, задачи, коммуникации.
- Маркетолог: источники лидов, кампании, UTM, формы.
- Поддержка/аккаунт-менеджер: сопровождение клиента, обращения.
- Интеграции/сервисы: системные учетные записи для вебхуков/API.

## 2. Бизнес-процессы и сценарии
- Приём лидов: формы/лендинги/реклама, распределение, обогащение данных.
- Работа с воронкой: стадии, автоматизации, напоминания, SLA по отклику.
- Коммуникации: звонки, письма, мессенджеры, SMS/push; лог активности.
- Сделки и счета: предложения, расчёты, платежи/статусы.
- Поддержка: тикеты/заявки, статусы, очереди.
- Отчёты и аналитика: воронка, конверсия, эффективность каналов, KPI.

## 3. Функциональные требования
- Управление лидами/контактами/компаниями, связка с задачами и коммуникациями.
- Настраиваемые воронки и поля, сегментация и фильтры.
- Планировщик задач/напоминаний, календарь, SLA/таргет отклика.
- История взаимодействий (omnichannel timeline), вложения/файлы.
- Автоматизации: правила, триггеры, шаблоны, рассылки.
- Импорт/экспорт данных, контроль дублей, мердж записей.
- Интеграции: вебхуки, API, телефония, почта, мессенджеры, платёжные сервисы.
- Настройки безопасности: роли/права, аудит, разграничение по отделам.

## 4. Нефункциональные требования
- Производительность: целевые SLO по отклику API/UI, целевой TPS.
- Надёжность: бэкапы, отказоустойчивость, RPO/RTO.
- Масштабируемость: горизонтальное масштабирование ключевых сервисов.
- Соответствие: требования к хранению персональных данных.
- Наблюдаемость: метрики, логи, трассировки, алерты.

## 5. Данные и модель
- Основные сущности: лид, контакт, компания, сделка, продукт, счёт, задача, тикет, коммуникация.
- Связи между сущностями (пример: контакт ↔ компания; сделка ↔ продукт/счёт; коммуникации ↔ задачи).
- Обязательные/кастомные поля, валидация, справочники.

## 6. API и интеграции
- REST/GraphQL конечные точки для CRUD сущностей.
- Вебхуки на события (создание/обновление/статусы сделок/задач/коммуникаций).
- Аутентификация для API: ключи, OAuth2, rate limits.

## 7. UX/UI
- Основные рабочие места: список лидов/сделок, карточка клиента, задача, лента взаимодействий.
- Требования к навигации, фильтрам, быстрому поиску, тегам.
- Набор базовых экранов: дашборды, отчёты, настройки, справочники.

## 8. Отчётность и метрики
- Конверсии по стадиям, скорость обработки, эффективность каналов.
- KPI менеджеров, загрузка, план/факт.
- Экспорт/доступ к BI-инструментам.

## 9. Риски и допущения
- Неполные/разрозненные источники данных.
- Ограничения интеграций (телефония, почта, мессенджеры).
- Ограниченный бюджет/сроки; приоритет MVP.

## 10. Открытые вопросы
- Обязательные интеграции? Какие уже есть системы.
- Политика хранения/удаления персональных данных.
- Нужен ли мобильный клиент или оффлайн-доступ.
